================================================================================
ΠΛΑΝΟ: Fragment-based rerun ανά tab στη σελίδα αποτελεσμάτων
================================================================================
Ημερομηνία: για μελλοντική αναφορά
Σκοπός: Με αλλαγή φίλτρων να ξανατρέχει μόνο ο κώδικας του ενεργού tab,
         όχι ολόκληρο το script (Streamlit @st.fragment).

================================================================================
ΠΡΟΒΛΗΜΑ
================================================================================
Στο Streamlit κάθε αλληλεπίδραση (π.χ. αλλαγή φίλτρου σε ένα tab) προκαλεί
full script rerun. Στη show_results_page (LOCAL_DEV/kyria/app_final.py,
περίπου γραμμές 4769–10300+) τρέχουν όλα τα tabs σε κάθε rerun (Σύνοψη,
Σύνολα, Καταμέτρηση, Κενά, ΑΠΔ, Παράλληλη, κ.λπ.), οπότε η εφαρμογή βαραίνει.

================================================================================
ΛΥΣΗ: @st.fragment ανά tab
================================================================================
Από την Streamlit 1.37 υπάρχει το st.fragment: όταν ο χρήστης αλλάζει widget
ΜΕΣΑ σε ένα fragment, ξανατρέχει ΜΟΝΟ η συνάρτηση του fragment, όχι ολόκληρο
το script.

  ΤΩΡΑ:    Φίλτρο αλλαγή --> Full rerun --> Όλα τα tabs
  ΜΕΤΑ:    Φίλτρο αλλαγή --> Fragment rerun --> Μόνο το tab

================================================================================
ΒΗΜΑΤΑ ΥΛΟΠΟΙΗΣΗΣ
================================================================================

1. Επιβεβαίωση έκδοσης Streamlit
   - Στο requirements.txt να προστεθεί (ή να ενημερωθεί) streamlit>=1.37
     ώστε να υπάρχει σίγουρα το st.fragment.

2. Αποθήκευση shared state σε session_state
   Στην ΑΡΧΗ της show_results_page (μετά τα βασικά precompute που ήδη γίνονται)
   να γράφονται σε st.session_state:
   - df → ήδη υπάρχει ως extracted_data (μπορεί να χρησιμοποιείται αυτό).
   - description_map, plafond_map, insurance_status_message, is_palios.
   - has_parallel, has_parallel_2017, has_multi (όπως υπολογίζονται τώρα).
   - view_exports / register_view: η λογική να δείχνει σε
     st.session_state['results_view_exports'] και το register_view να γράφει
     εκεί, ώστε να είναι διαθέσιμο και σε fragment reruns (π.χ. για
     «Εξαγωγή πίνακα»).

   Έτσι στο fragment rerun τα fragments διαβάζουν αυτά τα δεδομένα από
   session_state χωρίς να έχει ξανατρέξει ολόκληρη η show_results_page.

3. Εξαγωγή λογικής ανά tab σε συναρτήσεις-fragments
   Για ΚΑΘΕ κύριο tab (Σύνοψη, Σύνοψη AI Beta, Σύνολα, Καταμέτρηση, Κενά,
   ΑΠΔ, Παράλληλη, Παράλληλη 2017+, Πολλαπλή, Περισσότερα):

   - Εξαγωγή: το περιεχόμενο που τώρα τρέχει μέσα στο "with tab_xxx:" να μπει
     σε συνάρτηση, π.χ. _render_summary_tab(), _render_totals_tab(), … (ή να
     αξιοποιηθούν υπάρχουσες όπως render_totals_tab).
   - Fragment: η συνάρτηση να διακοσμηθεί με @st.fragment και να διαβάζει df
     (και ό,τι άλλο χρειάζεται) από st.session_state.
   - Κλήση: μέσα στο show_results_page, μετά το st.tabs(...), να παραμένει
     "with tab_xxx:" αλλά μέσα να καλείται ΜΟΝΟ η αντίστοιχη fragment-συνάρτηση,
     π.χ. _render_totals_tab().

   Τα tabs που ήδη κάνουν κλήση σε helper (π.χ. render_totals_tab) μπορούν να
   γίνουν wrapper fragments που διαβάζουν από session_state και καλούν το helper.

4. Keys widgets
   Τα widgets μέσα σε κάθε tab έχουν ήδη key=... (π.χ. key_prefix="totals",
   key="filter_taimeio"). Να παραμείνουν ΜΟΝΑΔΙΚΑ ανά tab ώστε να μην
   συγκρούονται όταν τρέχει μόνο ένα fragment. Αν χρειαστεί, να προστεθεί
   prefix ανά fragment (π.χ. totals_, gaps_) όπου δεν υπάρχει ήδη.

5. Tab «Περισσότερα» και nested tabs
   Το tab «Περισσότερα» έχει υπο-tabs (Κύρια Δεδομένα, Αποζημίωση, Παράρτημα).
   - Επιλογή Α: ένα fragment για όλο το «Περισσότερα» (αλλαγή φίλτρου σε
     οποιοδήποτε υπο-tab ξανατρέχει μόνο αυτό το fragment).
   - Επιλογή Β: ένα fragment ανά υπο-tab, αν η δομή του Streamlit το επιτρέπει.
   Προτείνεται Επιλογή Α για απλότητα.

6. Τάξη εμφάνισης και πρώτο full run
   Στο ΠΡΩΤΟ full run (όταν ανοίγει η σελίδα αποτελεσμάτων) θα τρέχουν όλα τα
   fragments μία φορά (γιατί μπαίνουμε σε όλα τα "with tab_xxx:" και καλούμε
   τις fragment-συναρτήσεις). Αυτό είναι αναμενόμενο.
   Στα επόμενα reruns, όταν ο χρήστης αλλάζει φίλτρο ή widget ΜΕΣΑ σε ένα tab,
   θα ξανατρέχει ΜΟΝΟ το αντίστοιχο fragment.
